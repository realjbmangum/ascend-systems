Technical SEO review of ascendsystems.ai measured against the goal of "humming for target keywords in 3–6 months." Findings prioritized P0–P3 with a week-by-week 90-day plan. TL;DR & Overall SEO Grade

Overall Grade: D+

The site is well-designed, well-written, and clearly positioned — but it is functionally invisible to Google. It is a client-side React SPA with no server-side rendering, no robots.txt, no sitemap.xml, no canonical tags, no structured data, no per-route HTML, and a soft-404 pattern that returns 200 OK for every URL including ones that don't exist. Every page Googlebot crawls today returns the same generic title and meta description from index.html. The site does not yet meaningfully exist in the index.

Top 3 P0s, in order:

  1. Add pre-rendering / SSR. Without it, none of the per-page SEO components, H1s, or copy are visible to crawlers. Nothing else in this audit matters until this is fixed.
  2. Ship robots.txt + sitemap.xml. Both 404 today. Google has no entry point.
  3. Verify Google Search Console + Bing Webmaster Tools. Currently flying blind. Cannot diagnose what's actually indexed.

Brian's "humming in 3–6 months" goal is achievable. Local-intent terms like Charlotte custom software developer, Charlotte fractional CTO, and Charlotte AI consultant have low keyword difficulty and the closest local competitor (Bry AI) is the one to beat. But the technical foundation has to come first. If we fix the P0s in Weeks 1–2, the on-page work in Weeks 3–6 starts compounding, and Weeks 7–12 is where rankings show up.

Technical Findings

The Big One: SPA With No Server Rendering

The site is built with React 19 + Vite 8 + react-router-dom 7, served as static HTML+JS from Cloudflare Pages. index.html ships exactly one body element: <div id="root"></div> (verified at /Users/jbm/new-project/site-ai-tech-co/index.html:14). All routing, all content, all per-page <title> and <meta> updates happen client-side via document.title writes in src/components/SEO.tsx.

Modern Googlebot does eventually render JS — but rendering is a separate, slower, lower-priority pass than HTML crawling. For a brand-new domain with zero backlinks, expect the index queue for JS-rendered content to be measured in weeks, not hours. Bingbot, DuckDuckBot, and most AI crawlers (ClaudeBot in particular) render JS far less aggressively or not at all. Every social preview (Slack, iMessage, LinkedIn, Twitter, Discord) reads only the static HTML.

What Googlebot, Bingbot, and AI crawlers see for every URL on the site right now:

<title>Ascend Systems | Custom Software Development & AI Solutions | Charlotte NC</title>
<meta name="description" content="Custom software development, AI integrations, and business automation
for mid-market companies. Ship your next product in weeks, not months. Based in Charlotte, NC." />

No H1. No body content. No internal links. No schema. No canonical. Identical for /, /services, /about, /contact, /portfolio, /portfolio/city-of-charlotte, and /tools/cost-calculator — all confirmed via curl against the live site on 2026-05-12.

Fix: Add pre-rendering. Three viable paths in priority order:

  1. vite-plugin-prerender or prerender-spa-plugin. Render each known route at build time to static HTML. No code rewrite. Cost: 2–4 hours. This is the recommended path given there are 8 public routes and case-study slugs are a known static list.
  2. Migrate to Astro with React islands. Brian's preferred stack per global CLAUDE.md. Gives proper SSR + static export, much better Core Web Vitals than the current 704 KB JS bundle, and aligns with Cloudflare Pages workflow. Cost: 1–2 days. Highest payoff long-term.
  3. Cloudflare Workers HTML injection. A Worker in front of Pages that injects the right <title> / <meta> / canonical / OG tags based on path. Cheapest, but doesn't put body content in the HTML — solves the meta-tag problem only. Use as a stopgap if (1) and (2) are not feasible this week.

AI-Bot Crawlability

The site is on Cloudflare. The known gotcha is that Cloudflare's "Block AI bots" managed rule will silently 403 GPTBot, ClaudeBot, and PerplexityBot regardless of what robots.txt says (see global memory note feedback_cloudflare_ai_bot_block.md). This matters for Ascend because the positioning is "AI-native solo operator" and the long game includes citation by AI search tools.

Action: Log into Cloudflare → ascendsystems.ai zone → Security → Bots → confirm "Block AI bots" is OFF, and confirm no custom WAF rule targets the AI user-agents. Verify with:

curl -A "Mozilla/5.0 (compatible; GPTBot/1.1; +https://openai.com/gptbot)" \
     -sI https://ascendsystems.ai/ | head -3
curl -A "Mozilla/5.0 (compatible; ClaudeBot/1.0; +claudebot@anthropic.com)" \
     -sI https://ascendsystems.ai/ | head -3
