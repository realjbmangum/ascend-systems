**Decisions:**
- Used Cloudflare Email Workers (MailChannels) instead of SendGrid for email
- Magic links expire in 15 minutes (stored in D1 with token, role, expires_at)
- Tasks have optional client_id and lead_id for context
- Email sequences use "trigger" field for enrollment grouping (e.g., "lead_welcome")
- 15-minute cron for drip processor instead of on-demand to reduce API calls
- Session cookies are HttpOnly + SameSite=Strict for security
- Project notes visibility is boolean flag, not role-based (simpler for now)

**Gotchas:**
- Stripe utility functions need nodejs_compat flag in wrangler.toml
- D1 schema can't use CREATE TABLE (would fail on existing tables) — migration used CREATE TABLE IF NOT EXISTS
- MailChannels requires SPF/DKIM DNS records on noreply@ascendsystems.ai domain
- Stripe webhook requires raw body for signature verification — use c.req.raw.text() not c.req.json()
- Magic link tokens must be URL-encoded when passed as query params
- Session tokens must be validated against expires_at datetime('now') comparison
